START HIERSamenvatting voor Beginners

Nog nooit met AI gecodeerd? Dit is alles wat je moet weten in 2 minuten. De rest van de gids gaat dieper op elk onderwerp in.

WAT IS HET?Claude Code is je AI-assistent

Claude Code is een tool die in je terminal draait. Je beschrijft in gewoon Nederlands (of Engels) wat je wilt, en Claude schrijft de code, bewerkt bestanden, draait tests, en doet git -- allemaal voor je. Het leest je hele project en begrijpt de samenhang.

HOE WERKT HET?Jij stuurt, Claude voert uit

Je typt opdrachten zoals je tegen een collega zou praten:

"Maak een login pagina met e-mail en wachtwoord"
"Fix de bug in het contactformulier"
"Schrijf tests voor de checkout functie"
"Maak een commit en open een PR"

Claude leest de relevante bestanden, maakt een plan, en voert het uit. Jij reviewt het resultaat.

DE 5 KERNCONCEPTENDit is alles wat je nodig hebt

1. CLAUDE.md -- Een tekstbestand in je project met instructies voor Claude. "Gebruik TypeScript." "Test met Jest." "Comments in het Nederlands." Claude leest dit automatisch bij elke sessie. → Deel 3

2. Plan Mode -- Laat Claude eerst nadenken voor het begint te coderen. /plan of Shift+Tab twee keer. Claude analyseert, schetst een aanpak, en wacht op jouw goedkeuring. → Deel 8

3. Context = geheugen -- Claude onthoudt alles binnen een sessie, maar het geheugen is beperkt. Wissel van taak? Typ /clear. Lange sessie? Typ /compact. Dit is de #1 faalfactor als je het vergeet. → Deel 9

4. Git = je vangnet -- Commit na elke geslaagde wijziging. Als Claude iets kapotmaakt, draai je terug. Werk op branches, niet op main. Claude kan al je git-werk doen. → Deel 11

5. Tests & hooks = je bewaking -- Tests bewijzen dat code werkt. Hooks zijn automatische checks (lint, types) die fouten blokkeren voor ze in je code komen. Claude kan tests en hooks voor je schrijven. → Deel 7

EERSTE 10 MINUTENZo begin je
  1. Installeer: npm install -g @anthropic-ai/claude-code
  2. Open je project: cd mijn-project && claude
  3. Genereer CLAUDE.md: typ /init
  4. Stel een vraag: "Leg uit wat dit project doet"
  5. Maak iets: "Voeg een dark mode toggle toe"
  6. Commit: "Commit deze wijzigingen met een goede message"
HET GROTE VERSCHILVan "vibe coding" naar engineering

Vibe coding: prompt → plak → hoop dat het werkt → als het breekt, opnieuw promten.

AI-engineered: instructies geven → plan laten maken → reviewen → testen → committen. Elke fout wordt een regel die voorkomt dat het opnieuw gebeurt.

Het verschil is niet het model of de tool -- het is jouw systeem eromheen. Deze gids helpt je dat systeem stap voor stap op te bouwen.

Voel je niet overweldigd

Je hoeft niet alles tegelijk te leren. Begin met CLAUDE.md + /plan + /clear + committen. Voeg skills, hooks en agents toe wanneer je er klaar voor bent. Het leerpad in de Starter Kit geeft je een 10-weken roadmap.
