DEEL 11Git-discipline

AI kan regressies introduceren. Commit na elke voltooide taak. Niet elk uur -- na elke geslaagde wijziging. Terugdraaien wordt triviaal.

Git basics -- de essentiële workflow

Of je nu solo werkt of in een team: deze workflow voorkomt problemen.

1. Werk altijd op een branch, nooit op main

main (of master) is je stabiele versie. Elke wijziging begint op een eigen branch.

# Nieuwe branch aanmaken en ernaar switchen
git checkout -b feature/login-page

# Of laat Claude het doen
"Maak een nieuwe branch voor de login pagina"

2. Commit regelmatig met duidelijke messages

Kleine, frequente commits. Elke commit is een herstelpunt als iets misgaat.

# Handmatig
git add login.html styles.css
git commit -m "Add login page layout and styling"

# Of laat Claude het doen (schrijft betere messages)
"Commit deze wijzigingen met een goede message"
Let op

Commit nooit gevoelige data (API keys, wachtwoorden, .env bestanden). Gebruik .gitignore en een pre-commit hook voor secret detection.

Specifieke git add vs git add .

Bij AI-development is selectief toevoegen essentieel. Claude maakt soms onverwachte bestanden aan -- debug files, tijdelijke output, IDE-configuratie. Met git add . neem je dat allemaal mee zonder het te zien.

# GOED: specifieke bestanden
git add src/components/hero-section/index.js
git commit -m "Add hero section component"

# FOUT: alles toevoegen
git add .
# Pakt onbedoeld mee: debug files, IDE config, onafgemaakt werk
Commit-ritme

Commit na elke geslaagde wijziging. Niet elk uur, niet aan het eind van de dag -- na elke wijziging die werkt. Elke commit is een herstelpunt. Als Claude iets kapotmaakt in de volgende stap, draai je terug naar de laatste werkende staat.

3. Push je branch naar GitHub

Push maakt je werk beschikbaar voor anderen (en is een backup).

# Eerste keer pushen van een nieuwe branch
git push -u origin feature/login-page

# Daarna volstaat
git push

4. Open een Pull Request (PR)

Een PR is een verzoek om je branch samen te voegen met main. Hier vindt code review plaats.

# Laat Claude een PR aanmaken
"Open een PR met een samenvatting van de wijzigingen"

# Of via gh CLI
gh pr create --title "Add login page" --body "..."
Pro tip: Draft PRs

Laat Claude draft PRs maken. Laag risico -- review alles voordat het "ready" wordt.

5. Review, merge, en opruimen

Na goedkeuring: merge de PR naar main. Verwijder de branch.

# Na merge: lokale branch opruimen
git checkout main
git pull
git branch -d feature/login-page

CI/CD integratie

Claude Code draait niet alleen lokaal. Met GitHub Actions en GitLab CI/CD automatiseer je reviews en triage.

Automatische code reviewClaude reviewt elke PR automatisch op kwaliteit en securityGitHub Code Review
Issue triageNieuwe issues automatisch labelen, prioriteren, toewijzenGitHub Actions
PR van bug report@Claude in Slack met bug → PR terugSlack integratie
# Claude in een CI pipeline (non-interactief)
claude -p "review this PR for security issues" \
  --output-format json \
  --max-budget-usd 2
Let op

CI-gebruik verbruikt API-credits. Gebruik --max-budget-usd om kosten te begrenzen en --fallback-model haiku voor goedkopere runs.
