DEEL 25Dagelijkse Tips & Tricks

Praktische features die je dagelijks sneller maken. De meeste hiervan ontdek je pas na weken werken -- hier staan ze alvast.

Afbeeldingen en screenshots gebruiken

Claude Code is multimodaal -- je kunt afbeeldingen meesturen in je prompt. Enorm handig voor UI-bugs, design reviews, of foutmeldingen.

MethodeHoe
PlakkenCtrl+V in de chat -- plakt een screenshot van je klembord
BestandspadSleep een bestand naar de terminal, of typ het pad: /Users/jij/screenshot.png
@ referentie@screenshot.png als het bestand in je project staat
URLPlak een afbeeldings-URL direct in je prompt
 
# Voorbeeldprompts met afbeeldingen
"Hier is een screenshot van de bug [Ctrl+V] -- fix de CSS"
"Maak deze Figma mockup @designs/homepage.png na in React"
"Wat gaat er mis in deze foutmelding? [plak screenshot]"

Voice input (spraak)

Claude Code ondersteunt push-to-talk spraakherkenning. Houd de Spatiebalk ingedrukt in de chat om te dicteren in plaats van te typen.

Wanneer handig

Bij lange uitleg van wat je wilt, als je handen moe zijn, of als je sneller praat dan typt. Claude ontvangt de getranscribeerde tekst als gewone prompt.

Sessies hervatten en doorzoeken

Claude Code onthoudt je sessiegeschiedenis. Je hoeft niet elke keer opnieuw te beginnen.

ActieHoe
Laatste sessie hervattenclaude --resume of claude -r
Sessie kiezenclaude --resume toont een lijst met recente sessies
Sessie zoeken op tekstCtrl+R in Claude Code -- zoek door je promptgeschiedenis
Vorige prompt hergebruikenPijltje omhoog (↑) in de chat
Sessie exporteren/export -- slaat het hele gesprek op als Markdown
 
# Hervat je laatste sessie
claude --resume

# Hervat een specifieke sessie
claude --resume abc123

# Continue in een specifieke map
claude --resume --cwd /pad/naar/project

Stash -- werk even parkeren

Met Ctrl+S kun je je huidige invoer stashen (opzij zetten). Handig als je midden in een prompt zit maar even iets anders moet doen.

# Je bent een lange prompt aan het typen...
# Druk Ctrl+S om te stashen
# Doe iets anders
# Druk Ctrl+S weer om je gestashte tekst terug te halen

Undo -- wijzigingen terugdraaien

Als Claude een wijziging maakt die je niet bevalt, kun je die direct ongedaan maken:

ActieHoe
Laatste actie ongedaan makenEsc Esc (dubbel escape) of Ctrl+_
Terugspoelen in gesprek/rewind -- ga terug naar een eerder punt in het gesprek
Specifiek punt herstellen/rewind 3 -- ga 3 stappen terug
Tip

/rewind is als een "undo" voor het hele gesprek. Claude vergeet alles na dat punt en je kunt een andere richting inslaan. Veel krachtiger dan handmatig terugdraaien.

Background tasks -- multitasken

Je kunt een lopende taak naar de achtergrond sturen zodat je ondertussen verder kunt werken.

ActieHoe
Taak naar achtergrondCtrl+B terwijl Claude werkt
Overzicht van taken/tasks
Taak terughalen/tasks en selecteer de taak
 
# Start een langlopende taak
"Refactor alle API calls naar de nieuwe SDK"
# Druk Ctrl+B -- taak gaat naar achtergrond
# Je kunt nu een nieuwe prompt typen
"Wat is de status van de migratie naar v2?"
# Check later: /tasks

Model wisselen mid-gesprek

Je hoeft niet vast te zitten aan een model. Wissel on-the-fly:

ActieHoe
Model picker openenMeta+P (Cmd+P op Mac)
Fast mode toggleMeta+O (Cmd+O op Mac) of /fast
Thinking toggleMeta+T (Cmd+T op Mac)
Effort level aanpassenIn model picker: pijltjes links/rechts
Wanneer wisselen

Opus voor complexe architectuurbeslissingen en moeilijke bugs.
Sonnet voor dagelijks werk -- snel en goed genoeg.
Haiku voor simpele taken, formatting, of als fallback.
Fast mode voor snellere output bij eenvoudige vragen.

Externe editor voor lange prompts

Voor complexe, lange prompts kun je je favoriete editor gebruiken:

# Druk Ctrl+G om je $EDITOR te openen (vim, nano, VS Code)
# Schrijf je prompt daar met alle ruimte
# Sla op en sluit -- de tekst wordt als prompt verstuurd

# Stel je editor in via environment variable
export EDITOR="code --wait"   # VS Code
export EDITOR="vim"            # Vim
export EDITOR="nano"           # Nano

Slim werken met @ referenties

Met @ verwijs je direct naar bestanden, mappen of URLs in je prompt. Claude leest ze automatisch mee.

# Verwijs naar een bestand
"Refactor @src/utils/api.ts naar async/await"

# Verwijs naar een hele map
"Bekijk @src/components/ en verbeter de TypeScript types"

# Verwijs naar meerdere bestanden
"Vergelijk @old-api.ts met @new-api.ts en documenteer de verschillen"

# Verwijs naar een URL
"Implementeer de API volgens @https://api-docs.example.com/v2"

Notificaties bij lange taken

Configureer een notificatie-hook zodat je een melding krijgt als Claude klaar is:

// .claude/settings.json
{
  "hooks": {
    "Notification": [{
      "matcher": "",
      "hooks": [{
        "type": "command",
        "command": "osascript -e 'display notification \"$CLAUDE_NOTIFICATION\" with title \"Claude Code\"'"
      }]
    }]
  }
}

Nu krijg je een macOS-melding als Claude klaar is met een langlopende taak -- handig als je ondertussen in een andere app werkt.

Diagnose en troubleshooting

ProbleemOplossing
Claude doet raar of is traag/doctor -- checkt installatie, API-verbinding, configuratie
Context vol / verward/clear om context te wissen, of /compact om samen te vatten
Plugin werkt niet/reload-plugins of claude mcp list voor MCP-status
Wil weten wat iets kost/cost voor huidige sessie, /usage voor totaalverbruik
Foute wijziging gemaaktEsc Esc voor undo, /rewind voor gesprek terugdraaien
Weet niet welke versie/status toont versie, model, account, connectiviteit
Wil alles resetten/clear + nieuw gesprek starten

Handige one-liners

# Snel een bestand laten reviewen
claude -p "Review dit bestand op bugs en security issues" < src/auth.ts

# Screenshot analyseren
claude -p "Beschrijf wat je ziet" < screenshot.png

# Git diff laten reviewen
git diff | claude -p "Review deze wijzigingen, focus op bugs"

# Meerdere bestanden tegelijk
cat src/api/*.ts | claude -p "Zijn er inconsistenties in deze API calls?"

# Output direct naar bestand
claude -p "Genereer een .gitignore voor een Node.js project" > .gitignore

# JSON output voor scripting
claude -p "Lijst alle TODO's in dit project" --output-format json
