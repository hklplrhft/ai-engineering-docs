DEEL 18Veelgemaakte Fouten

FOUT: De keukenlasessie

Taak A → iets anders → terug naar A. Context vol met irrelevante info.

FIX

/clear tussen ongerelateerde taken.

FOUT: Eindeloos corrigeren

Fout → correctie → nog steeds fout → correctie. Context vervuild met mislukte pogingen.

FIX

Na 2 mislukte correcties: /clear + betere prompt. Of Esc Esc om terug te spoelen.

FOUT: Over-specified CLAUDE.md

Te lang → Claude negeert de helft. Belangrijke regels verdrinken in ruis.

FIX

Meedogenloos snoeien. "Maakt Claude fouten zonder deze regel?" Zo niet → verwijder of verplaats naar ref-doc.

FOUT: Vage prompts

"Fix de bug" = 5-8 rondes. Specifieke prompt met error output = 1-2 rondes.

FIX

Plak exacte error, stack trace, bestandsnaam, verwacht gedrag.

FOUT: Altijd Opus

1.67x duurder. Over een maand: honderden euro's verschil.

FIX

Sonnet default, Opus alleen voor complexe taken. /model om te wisselen.

FOUT: Nooit committen

AI introduceert regressie, je kunt niet terug.

FIX

Commit na elke voltooide taak. Laat Claude de message schrijven.

FOUT: Blind vertrouwen

Claude levert code die er goed uitziet maar edge cases mist. Je shipt zonder te testen.

FIX

Geef altijd verificatiecriteria mee: tests, verwachte output, screenshots. Als je het niet kunt verifieren, ship het niet.

FOUT: Eindeloos laten zoeken

"Onderzoek dit" zonder scope. Claude leest honderden bestanden, vult je context.

FIX

Beperk de scope ("bekijk alleen src/auth/") of gebruik een subagent zodat de zoektocht je main context niet vervuilt.

FOUT: Code committen die je niet begrijpt

Claude genereert iets complex, je merged het zonder het te doorgronden.

FIX

Begrijp elke wijziging voor je merged. Vraag Claude: "Leg uit wat deze code doet en waarom." Als je het niet kunt uitleggen, commit het niet.
