START HIERContext & Doel

De rest van de monorepo (Astro apps, packages/ui, packages/api-new) is al grotendeels TypeScript en goed gestructureerd. Zoom.nl is de uitzondering:

598 JavaScript bestanden

Terwijl alle andere apps TypeScript-first zijn. Geen type safety, geen compiler die fouten vangt.

0 tests

Geen vangnet. Elke AI-wijziging is een gok -- je weet pas in productie of het werkt.

10 bestanden >300 regels

De foto detail pagina is 675 regels. Mixed concerns: data, logica, rendering in een bestand.

60+ hardcoded URLs

Geen Single Source of Truth. AI gaat URLs opnieuw hardcoden omdat er geen config is.

DOELDe codebase zo inrichten dat een AI moeilijk dingen kapot kan maken

Dit plan brengt zoom-nl stap voor stap in lijn met de rest van de monorepo. Niet herschrijven -- verbeteren. Zodat AI-assistenten dezelfde patronen kunnen volgen die al werken in de andere apps.

ACHTERGRONDDe 5 Bouwblokken van zoom-nl

Elk bouwblok heeft een specifieke rol. Na het opsplitsen (fase 3) kan een AI alleen het relevante bestand lezen en aanpassen.

Paginapages/ -- een URL in de browser. Haalt data op, rendert components. Alleen lijm.Voorbeeld: pages/index.js
Componentsrc/components/ -- herbruikbaar stuk UI. Weet hoe iets eruitziet, niet waar data vandaan komt.Voorbeeld: HeroSection
Hooksrc/hooks/ -- herbruikbare logica zonder UI. State, side effects, berekeningen. Begint met use.Voorbeeld: usePhotoLike
Utilsrc/utils/ -- pure functies zonder state. Input erin, output eruit. Overal bruikbaar.Voorbeeld: getImage()
Constantesrc/config/ -- waarden die niet veranderen. SSOT: als het op 3+ plekken voorkomt, hoort het hier.Voorbeeld: SITE_URL

FASE 0Guard Rails 1-2 DAGEN

Geen code-wijzigingen nodig. Dit is de "dag 0" setup die alles daarna veiliger maakt.

0.1
AGENTS.md uitbreiden met off-limits en patronen

Het huidige bestand is 25 regels en mist concrete regels. Voeg off-limits bestanden en patronen voor nieuwe code toe.

OFF-LIMITSBestanden die een AI niet mag wijzigen
- src/services/authentication.js  -- nooit zonder security review
- src/services/resources.js       -- token refresh, kritiek pad
- middleware.js                    -- raakt elke request
- src/services/captcha.js          -- reCAPTCHA verificatie
PATRONENHoe nieuwe code eruit moet zien
- Nieuwe pagina:     pages/_template.js als basis
- Nieuwe hook:       src/hooks/<naam>.js
- Nieuwe util:       src/utils/<naam>.js
- Nieuwe component:  src/components/<naam>/components/index.js
- Constanten:        src/config/site.js
- API URLs:          altijd via environment variables
"Breid apps/zoom-nl/AGENTS.md uit met off-limits bestanden (authentication.js, resources.js, middleware.js, captcha.js), patronen voor nieuwe code, en wat een AI niet mag doen. Lees het huidige bestand en docs/plan-zoom-nl-code-readiness.html voor de details."
Review door team -- klopt de off-limits lijst?
0.2
.env.example aanmaken

Er is geen .env.example. Een AI weet niet welke variabelen beschikbaar zijn en gaat URLs hardcoden.

NEXT_PUBLIC_SITE_URL=https://www.zoom.nl
NEXT_PUBLIC_IMAGE_HOST_DEV=https://mp-tst.reshift.nl/zoom/
NEXT_PUBLIC_IMAGE_HOST_PROD=https://mp.reshift.nl/zoom/
NEXT_PUBLIC_GTM_ID=GTM-P7PBC3
NEXT_PUBLIC_DFP_NETWORK_ID=116488029
"Maak een .env.example aan voor zoom-nl op basis van alle environment variabelen die in de code gebruikt worden. Zoek naar process.env in de hele app."
DevOps -- kloppen de variabelen met het Vercel dashboard?
0.3
Pagina template aanmaken

Een minimaal voorbeeld dat een AI kan kopieren. Voorkomt dat patronen worden overgenomen uit de 675-regel foto detail pagina.

"Maak pages/_template.js aan als minimaal voorbeeld voor nieuwe pagina's. Gebruik het bestaande getServerSideProps patroon uit andere pagina's."
0.4
Hooks instellen (lint, secrets, tests)
Kernvraag bij elke regel

"Wat als Claude dit negeert?" Vervelend = regel in AGENTS.md. Kapot = hook.

Pre-commit: lintFouten = commit geblokkeerd. Claude ziet fout en fixt zelf.DETERMINISTISCH
Pre-commit: secretsScan op hardcoded API keys, tokens, wachtwoorden.DETERMINISTISCH
Post-write: testsTests draaien automatisch na elke file write. Claude past aan bij falen.DETERMINISTISCH
Typ /hooks in Claude Code en configureer: - PreCommit: npx next lint --quiet - PreCommit: secret detection (grep op API keys) - PostWrite: npx jest --bail --findRelatedTests (na fase 4)
0.5
Skills aanmaken voor herhaalde taken

Taak 3x handmatig gedaan? Maak een skill in .claude/skills/.

"Maak een .claude/skills/new-page/SKILL.md skill aan voor het aanmaken van nieuwe zoom-nl pagina's. De skill moet _template.js kopieren, resource path aanpassen, en een build draaien."
0.6
Feedback loop: littekens-bestand aanmaken

Maak docs/refs/zoom-nl-littekens.md -- elke AI-fout wordt een regel die voorkomt dat het opnieuw gebeurt.

# Zoom.nl -- Littekens (geleerde lessen)
- image.fileName is camelCase, NIET image.filename
- creator.slug kan undefined zijn -- altijd null-checken

"Maak docs/refs/zoom-nl-littekens.md aan. Voeg de fouten toe die we al kennen: image.fileName is camelCase, creator.slug kan undefined zijn, dangerouslySetInnerHTML nooit zonder sanitizer."
Verificatie

AGENTS.md review door team, .env.example klopt met Vercel dashboard, hooks actief

Livegang

Geen deploy nodig. Dit zijn configuratiebestanden -- ze veranderen niets aan de applicatie. Merge naar develop, klaar.

FASE 3Grote Bestanden Opsplitsen 3-5 DAGEN

Waarom dit de grootste verbetering is

Bij een 675-regel bestand moet een AI het hele bestand lezen om iets kleins te wijzigen. Hogere kans op fouten, meer tokens, hogere kosten. Vuistregel: max 150 regels per bestand.

Belangrijk

Gebruik /plan voor elke split. Laat Claude eerst analyseren welke state, hooks en functies er zijn, en hoe ze opgesplitst kunnen worden. Akkoord geven, dan pas uitvoeren.

Fase 5: TypeScript migratie

Zoom.nl is de enige app in de monorepo die nog JavaScript is. TypeScript geeft een AI exacte type-informatie en vangt fouten op bij het bouwen.

5.1
TypeScript configuratie aanzetten
STAP 1allowJs: true

TS en JS bestaan naast elkaar. Geen big bang.

"Zet allowJs: true in de tsconfig.json van zoom-nl zodat TypeScript en JavaScript naast elkaar kunnen bestaan. Zorg dat de build nog werkt."
5.2
Type interfaces voor API responses
STAP 2Types voor API responsesHoogste waarde/effort

Definieer Photo, Creator, Gallery interfaces. Zelfs zonder bestanden te hernoemen geeft dit AI-assistenten structuur.

"Maak src/types/photo.ts aan met TypeScript interfaces voor Photo, Creator, Gallery en Challenge. Baseer de types op het daadwerkelijke gebruik in de codebase -- kijk naar de destructuring in de components."
Backend team -- kloppen de types met wat de API daadwerkelijk teruggeeft?
DOORLOPENDMigreer per feature

Nieuwe bestanden in TypeScript. Bestaande bestanden alleen hernoemen als je ze toch aanpast.

Verificatie

Tests: turbo run test --filter=zoom-nl slaagt, coverage >80% voor src/utils/
TypeScript: tsconfig.json actief, nieuwe bestanden zijn .ts/.tsx

Livegang

Geen risico. Tests voegen alleen bestanden toe. allowJs: true en type-bestanden veranderen niets aan runtime gedrag. Kan altijd live.
