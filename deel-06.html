DEEL 6Hooks — Harde Garanties DETERMINISTISCH

Shell-script dat automatisch draait. Anders dan CLAUDE.md-instructies en skills (die probabilistisch zijn -- de AI kan ze negeren): hooks zijn deterministisch — ze draaien altijd, zonder uitzondering.

Het fundamentele inzicht

Je kunt Claude niet vertrouwen om regels altijd te volgen. Taalmodellen zijn probabilistisch. Een pre-commit hook die tsc --noEmit draait weigert simpelweg de commit. Claude kan dit niet overrulen.

Begin met deze drie

1. Pre-commit: lint/type check -- Fouten = geen commit.
2. Post-write: test runner -- Claude krijgt testresultaat als feedback en fixt zelf.
3. Pre-commit: secret detection -- Scan op hardcoded API keys.

Kernvraag bij elke regel: "Wat als Claude dit negeert?" → Vervelend = regel. Kapot = hook.

Concreet voorbeeld

Zo ziet een hook-configuratie eruit in .claude/settings.json:

{
  "hooks": {
    "PreCommit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "npm run lint && npm run typecheck"
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "npm test -- --related"
          }
        ]
      }
    ]
  }
}

Wat gebeurt er? Bij elke commit draait automatisch lint + typecheck. Falen ze? Commit geblokkeerd. Na elke bestandswijziging draaien de gerelateerde tests. Faalt een test? Claude ziet de foutmelding en fixt het zelf.

Configureer via /hooks (interactief) of edit .claude/settings.json handmatig. Claude kan ook hooks voor je schrijven: "Maak een pre-commit hook die lint en types checkt."

Voorbeelden per stack

Het voorbeeld hierboven is voor een Node.js-project. Hier zijn de commando's voor andere stacks:

StackPreCommitPostToolUse
Next.js / Reactnpm run lint && npx tsc --noEmitnpm test -- --related
Flutterflutter analyze && dart format --set-exit-if-changed .flutter test

Gogo vet ./... && staticcheck ./...go test ./...

Bonus: desktop notificaties

Start een lange taak en wil je weten wanneer Claude klaar is? Stel een Notification hook in:

"Notification": [{
  "matcher": "",
  "hooks": [{
    "type": "command",
    "command": "osascript -e 'display notification \"Claude is klaar\" with title \"Claude Code\"'"
  }]
}]

SessionStart: project-check bij elke sessie

De signalen in je globale CLAUDE.md zijn probabilistisch -- Claude kan ze vergeten. Een SessionStart hook is deterministisch: het draait bij elke sessie, zonder uitzondering.

Het shell-script

Maak ~/.claude/hooks/project-check.sh aan:

#!/bin/bash
# ~/.claude/hooks/project-check.sh
# Checkt of de basis project-inrichting aanwezig is

warnings=""

# Check: is dit een git repository?
if [ ! -d ".git" ]; then
  warnings="${warnings}\n- Geen git repository gevonden. Overweeg: git init"
fi

# Check: heeft het project een CLAUDE.md?
if [ ! -f "CLAUDE.md" ]; then
  warnings="${warnings}\n- Geen CLAUDE.md gevonden. Stel voor: /init"
fi

# Check: is er een docs/refs/ map?
if [ ! -d "docs/refs" ]; then
  warnings="${warnings}\n- Geen docs/refs/ map. Maak aan bij eerste bugfix."
fi

# Check: staat .env in .gitignore?
if [ -f ".gitignore" ]; then
  if ! grep -q "^\.env" .gitignore 2>/dev/null; then
    warnings="${warnings}\n- .env staat niet in .gitignore -- secrets gevaar!"
  fi
else
  warnings="${warnings}\n- Geen .gitignore gevonden. Maak er een aan."
fi

# Output als systemMessage (alleen bij warnings)
if [ -n "$warnings" ]; then
  echo "PROJECT-CHECK: De volgende onderdelen ontbreken:"
  echo -e "$warnings"
  echo ""
  echo "Bied aan om ontbrekende onderdelen op te zetten."
fi

Configuratie in settings.json

Voeg toe aan je globale ~/.claude/settings.json (niet per project -- dit geldt voor alle projecten):

{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/.claude/hooks/project-check.sh"
          }
        ]
      }
    ]
  }
}
Wat gebeurt er?

Bij elke nieuwe Claude Code sessie draait het script automatisch. Als er iets ontbreekt, krijgt Claude een waarschuwing als systemMessage en biedt aan om het op te zetten. Als alles in orde is, geen output -- geen ruis.

Dit maakt de basis-inrichting uit de Starter Kit onmogelijk te vergeten. De probabilistische signalen in je globale CLAUDE.md (tests, skills, agents) groeien organisch -- de SessionStart hook garandeert het fundament.

Plugins

Plugins bundelen skills, hooks, agents en MCP servers in een installeerbaar pakket. Bekijk de marketplace via /plugin. Tip: installeer een code intelligence plugin voor je taal -- geeft Claude "go to definition" en automatische foutdetectie na edits.
