DEEL 3Het Instructiesysteem — De 5 Lagen

Het hart van AI-engineered development. Bouw op in deze volgorde.

LAAG 1Globale regels~/.claude/CLAUDE.md

Persoonlijke voorkeuren voor alle projecten. "Schrijf comments in het Nederlands." · "Gebruik hyphens, nooit em-dashes." · "Vraag bevestiging voor verwijderen."

Vuistregel: Geldt voor elk project → hier. Projectspecifiek → niet.

LAAG 2Projectregels./CLAUDE.md

Briefing voor je specifieke project. Automatisch geladen. Houd het compact -- in de praktijk blijkt 400+ regels te veel; ~100 regels werkt beter. Frontier modellen volgen ~150-200 instructies, minus ~50 voor Claude Code's systeemprompt. Je hebt ~100-150 regels ruimte.

Erin: Project overview, tech stack, kritieke regels, verwijzingen naar ref-docs, testcommando's.
Niet erin: Code style (→ linter/hook), code-voorbeelden (→ verouderen), afleidbare informatie.

Progressive disclosure -- kleine bestanden

CLAUDE.md is een inhoudsopgave, niet het hele boek. Stop details in aparte bestanden en verwijs ernaar. Claude laadt ze wanneer nodig.

Voorbeeld bestandsstructuur

mijn-project/
├── CLAUDE.md              ← kort: overzicht + verwijzingen
├── docs/
│   └── refs/
│       ├── api-regels.md  ← API conventies en valkuilen
│       ├── db-schema.md   ← kolomnamen, relaties, gotchas
│       └── deploy.md      ← deployment stappen
├── .claude/
│   ├── settings.json      ← permissions, hooks
│   ├── agents/            ← project agents
│   └── skills/            ← project skills
└── src/                   ← je code

Voorbeeld CLAUDE.md

# Mijn Project

## Overview
Webshop gebouwd met Next.js 14, Prisma ORM, PostgreSQL.

## Tech stack
- Frontend: React + Tailwind CSS
- Backend: Next.js API routes
- Database: PostgreSQL via Prisma
- Tests: Jest + React Testing Library

## Kritieke regels
- ALTIJD Prisma migrate gebruiken, nooit raw SQL voor schema changes
- Componenten in src/components/, pages in src/app/
- Zie docs/refs/api-regels.md voor API conventies
- Zie docs/refs/db-schema.md voor database valkuilen

## Commando's
- Test: npm test
- Lint: npm run lint
- Build: npm run build
- Dev: npm run dev

Quick start: draai /init om een starter CLAUDE.md te genereren.

LAAG 3Taakspecifieke regelsdocs/refs/

Je "littekens" -- lessen van productiefouten. Laden op aanvraag. Claude maakt fout → je schrijft regel → nooit meer herhaald.

# docs/refs/db-schema.md

## Valkuilen
- De kolom is `is_answer_ready`, NIET `answer_ready`
- Gebruik altijd soft deletes (deleted_at), nooit hard deletes
- Foreign keys MOETEN ON DELETE CASCADE hebben

## Naamconventies
- Tabellen: meervoud, snake_case (users, order_items)
- Kolommen: snake_case (created_at, is_active)
Hoe het werkt

CLAUDE.md zegt: "Zie docs/refs/db-schema.md voor database valkuilen." Claude leest dat bestand alleen wanneer hij met de database werkt. Zo blijft je hoofdbestand klein en je context schoon.

LAAG 4Persistent geheugenMEMORY.md + auto memory

Overleeft sessies. Bekende bugs, beslissingen, project IDs, procedures. Zonder dit begint elke sessie opnieuw.

Handmatig geheugen

/memory om CLAUDE.md memory bestanden te bewerken. Schrijf regels, beslissingen en context op die je wilt behouden.

Auto memory

Claude leert automatisch tijdens het werken. Build-commando's, debugging-inzichten, project-patronen -- worden opgeslagen zonder dat je iets hoeft te schrijven. Schakel in/uit via /memory.

Twee vormen, samen sterker

Auto memory vangt operationele kennis op (hoe te builden, welke tests draaien). Handmatig geheugen voor architectuurbeslissingen en regels die Claude moet volgen. Gebruik beide.

Handoff files. Beëindig je een sessie? Vraag Claude om HANDOFF.md te schrijven met: doel, wat geprobeerd, wat werkte, wat faalde, volgende stappen.

LAAG 5Uitvoerbare extensiesskills, agents, hooks

Code die Claude niet alleen leest maar uitvoert. Van "de AI helpt mij" naar "het systeem werkt voor mij." Vier bouwstenen:

SkillsHerbruikbare instructieset (recept). Claude volgt de stappen in je huidige sessie.Taak 3x handmatig gedaan? Maak een skill.
AgentsZelfstandige Claude-instantie met eigen context. Draait parallel, eigen focus.Werk isoleren, parallel reviewen, bulk taken.
HooksShell-script dat automatisch draait. Deterministisch -- kan niet genegeerd worden.Alles wat kapot gaat als Claude het negeert.
TestsBewijs dat code werkt. Vangt regressies op voor ze in productie komen.Elke functie die fout mag gaan = een test.

Kernonderscheid: Skills en agents zijn probabilistisch -- Claude kan afwijken. Hooks en tests zijn deterministisch -- ze blokkeren altijd. Gebruik beide lagen samen.
