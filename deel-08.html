DEEL 8Plan Mode -- Denken Voor Doen

Het mechanisme achter de 50% voorbereiding. Claude kan alles analyseren maar niets wijzigen.

Activeren: Shift+Tab twee keer, of /plan. Plan bewerken: Ctrl+G opent het plan in je editor.

De vier fasen

  1. Explore -- Plan Mode aan. Claude leest bestanden, begrijpt de structuur.
  2. Plan -- Claude schetst een stappenplan (wat, welke volgorde, waarom).
  3. Implement -- Terug naar Normal Mode. Claude codeert volgens het plan.
  4. Commit -- "Commit met een beschrijvende message en open een PR."

Geef verificatiecriteria mee

De #1 tip van Anthropic

Claude presteert dramatisch beter als het zijn eigen werk kan verifiëren. Geef tests, screenshots, of verwachte output mee.

Vaag"Implementeer email validatie"Claude kan niet checken of het werkt
Specifiek"Schrijf validateEmail. user@example.com = true, invalid = false. Draai de tests na implementatie."Claude verifieert zelf

Laat Claude je interviewen

Voor grotere features: laat Claude eerst vragen stellen over dingen die je misschien niet hebt overwogen.

"Ik wil [korte beschrijving] bouwen.
 Interview me over technische implementatie, edge cases,
 UI/UX en tradeoffs. Stel door tot we alles hebben besproken.
 Schrijf dan een spec naar SPEC.md."

→ Start daarna een frisse sessie met /clear
→ "Implementeer de spec in SPEC.md"

Gebruik @-referenties

Verwijs direct naar bestanden met @ in je prompt -- Claude leest ze meteen:

"Leg de logica uit in @src/utils/auth.js"
"Wat is de structuur van @src/components?"
"Vergelijk @file1.js en @file2.js"

Wanneer Plan Mode: Complexe refactoring, multi-file features, onbekende codebases. Alles wat meer dan een bestand raakt. Wanneer overslaan: Als je de diff in een zin kunt beschrijven.

Veelgemaakte fout

Claude meteen laten coderen → kan code opleveren die het verkeerde probleem oplost. Plan Mode voorkomt uren fixen.

Plan opslaan: Vraag Claude om het plan naar plan.md te schrijven. Wordt werkgeheugen dat sessies overleeft.
