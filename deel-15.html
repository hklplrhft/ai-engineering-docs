DEEL 15Single Source of Truth

3-plekken regel: Staat dezelfde waarde op 3+ plekken? Maak een SSOT. Eén bron, de rest verwijst.

Waarom dit belangrijk is

Zonder SSOT verandert Claude de waarde op 2 van de 3 plekken en vergeet de derde. Bugs die je uren kosten om te vinden.

In je code

// GOED: één bron
export const CONFIG = { maxUpload: 10485760 }
// Overal verwijzen naar CONFIG.maxUpload

// FOUT: dezelfde waarde op meerdere plekken
const MAX_UPLOAD = 10485760;       // in upload.js
const UPLOAD_LIMIT = 10485760;     // in validation.js
"Maximaal 10MB"                    // in error message
// → Claude past er 2 aan, vergeet de derde

In je instructies

// GOED: definieer in docs/refs/, verwijs vanuit CLAUDE.md
# CLAUDE.md
"Zie docs/refs/api-regels.md voor API conventies"

// FOUT: dezelfde regel in CLAUDE.md, in een skill, en in een agent
// → regels raken uit sync, Claude krijgt tegenstrijdige instructies

In je project

ConfiguratieEén config bestand, overal importerenNiet kopiëren naar meerdere bestanden
ConstantenEén constants.js/ts, exporteer allesNiet dezelfde waarde in 3 bestanden
Types/interfacesEén types bestand per domeinNiet herdefinieren per component
Claude-regelsDefinieer in docs/refs/, verwijs vanuit CLAUDE.mdNiet dezelfde regel op meerdere plekken
Vuistregel

Als je dezelfde waarde wilt wijzigen, moet je dat op precies 1 plek hoeven doen. Moet je op meerdere plekken zoeken? Dan heb je een SSOT-probleem.
